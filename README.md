# 하나 더 (Hana More) - 갓생스코어 앱 데모 🌱

**"소득 증빙이 어려운 긱워커를 위한 블록체인 기반 대안신용평가 플랫폼"**

> 본 프로젝트는 하나은행 공모전 출품작의 프론트엔드/도메인 로직 프로토타입(Demo Ver.1)입니다. 

---

## 🎯 프로젝트 개요
**하나 더 (Hana More)**는 프리랜서, 긱워커 등 정규 소득 증빙이 어려워 기존 금융권의 신용평가에서 소외된 계층을 위해 기획된 **대안신용평가 플랫폼**입니다. 사용자의 다양한 일상 활동과 금융 패턴을 4가지 핵심 행동 피처(Feature)로 수집하고, 이를 머신러닝 가중치 기반으로 평가하여 자체적인 **'갓생점수(God-Saeng Score)'**를 산출합니다.

## ✨ 주요 기능 및 비즈니스 로직

### 1. 4가지 행동 피처(Feature) 수집 및 분석
앱 내의 미션과 금융 데이터를 통해 사용자의 16개 세부 지표를 수집합니다.
- **Type A (생활 루틴):** 기상 시간, 수면 규칙성, 미션 달성률 등 일상의 성실도
- **Type B (일/소득):** 포트폴리오 업데이트, 소득 발생 빈도, 업무 완료 인증 (소득 지속가능성)
- **Type C (소비 행동):** 소비 패턴 규칙성, 심야 충동 결제 비율, 잔고 유지 여부 (소비 통제력)
- **Type D (개인 ESG):** 대중교통 이용, 에너지 절약, 운동 및 기부 활동 등

### 2. 실시간 갓생점수 산출 알고리즘 (ML 기반 모의 엔진)
- XGBoost 트리 모델 구조를 차용한 **가중치 기반 계산식**을 클라이언트 단에 구현했습니다.
- 분기 단위 누적 스코어와 현재 행동 변화율을 복합적으로 계산하여 신용 등급을 결정합니다.
- 단순 합산이 아닌 **SHAP Value(설명 가능한 AI)** 개념을 도입해, 유저에게 "어떤 행동이 점수를 올렸는지(또는 깎았는지)"를 투명하게 제공합니다.

### 3. 블록체인 기반 데이터 무결성 (Anti-Tamper Pipeline)
- 수집된 사용자 행동 데이터는 클라이언트에서 `Keccak256` 암호화 해싱을 거쳐 기록됩니다. 
- 현재 데모 버전에서는 **모의 파이프라인(Mock Pipeline)**으로 구현되어 있으며, 기획상 훼손 불가능한 점수 증명과 스마트 컨트랙트 연동을 대비하여 설계되었습니다.

## 🛠 기술 스택 (Tech Stack)

### 프론트엔드 및 아키텍처
- **Framework:** React Native (Expo)
- **Language:** TypeScript 
- **State Management:** Zustand (Immer를 활용한 불변성 집중 관리)
- **Architecture:** **Clean Architecture** 원칙 반영 (Domain, Application, Infrastructure, Presentation 계층의 명확한 분리)
  - `src/domain`: 핵심 계산 로직 (순수 함수) 및 엔티티
  - `src/application`: Zustand Store 및 상위 비즈니스 훅 (useMLEngine 등)
  - `src/infrastructure`: Mock DB 연동, 블록체인 해싱 관련 인프라 로직
  - `src/presentation`: UI 컴포넌트 및 앱 네비게이션

## 🚀 구동 방식 (Getting Started)

본 프로젝트는 Expo 기반으로 작성되었습니다.

### 1. 의존성 패키지 설치
```bash
npm install
# 또는
yarn install
```

### 2. 앱 실행 (개발 서버 열기)
```bash
npm run start
# 또는
yarn start
```
- 명령어 실행 후 터미널에 나타나는 **QR 코드**를 스마트폰의 카메라(iOS)나 Expo Go 앱(Android/iOS)으로 스캔하여 실기기에서 테스트할 수 있습니다.
- 키보드에서 `i`를 눌러 iOS 시뮬레이터에서 실행하거나, `a`를 눌러 안드로이드 에뮬레이터에서 실행할 수도 있습니다.

## 💡 데모 시나리오 안내
이 데모 앱은 실제 서버 통신 없이 **인메모리(Mock DB) 시스템**으로 동작합니다. 
코드를 실행하면 다음과 같은 파이프라인 흐름을 테스트해 볼 수 있습니다:
1. 유저가 '기상 인증'이나 '포트폴리오 등록' 등 특정 미션을 수행합니다 (`CompleteMission`).
2. 프론트엔드는 데이터를 수합 후 내부적으로 `Keccak256` 해시 식별자를 생성합니다.
3. Zustand 스토어의 `missionStore`가 파이프라인을 작동시켜 미션 내역에 기록합니다 (`missionPipeline.ts`).
4. `godScoreStore`가 최신 데이터를 반영해 **갓생점수**를 즉시 재계산 및 렌더링합니다.
